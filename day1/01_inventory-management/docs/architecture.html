useFilters()

Singleton filter state shared across all views. The single source of truth for all API query parameters.

  • selectedPeriod / selectedLocation / selectedCategory / selectedStatus (refs)
  • getCurrentFilters() — builds query param object
  • hasActiveFilters — computed boolean
  • resetFilters() — resets all to "all"

useI18n()

Full internationalisation. Locale persisted in localStorage. Switches UI text and currency symbol.

  • currentLocale ref (en | ja)
  • t(key) — translate string with fallback
  • currentCurrency — USD or JPY
  • translateProductName / Customer / Warehouse

useAuth()

Demo user state. No real auth — isAuthenticated is always true. User profile switches with locale.

  • currentUser computed (name, email, department)
  • isAuthenticated (always true)
  • logout() — alert only (demo)
  • getInitials(name)
API Endpoints
Method Path Query Params Response
GET /api/inventory warehouse, category List of InventoryItem
GET /api/inventory/{id} — Single InventoryItem or 404
GET /api/orders warehouse, category, status, month List of Order — month supports quarters (Q1-2025)
GET /api/orders/{id} — Single Order or 404
GET /api/demand — List of DemandForecast
GET /api/backlog — List of BacklogItem (has_purchase_order computed)
GET /api/dashboard/summary warehouse, category, status, month KPI stats: inventory value, low stock, pending orders, backlog
GET /api/spending/summary — Total costs by category with % change vs prior period
GET /api/spending/monthly — Jan–Dec monthly breakdown (procurement, operational, labor, overhead)
GET /api/spending/categories — Category breakdown: Raw Materials, Components, Equipment, Consumables
GET /api/spending/transactions — List of Transaction records
GET /api/reports/quarterly — Q1–Q4 2025: revenue, orders, fulfillment rate, avg order value
GET /api/reports/monthly-trends — Monthly order count, revenue, delivered count
POST /api/purchase-orders Body: PurchaseOrder Created PurchaseOrder (frontend only — not yet in backend)
