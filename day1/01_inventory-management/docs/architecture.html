Request / Filter Data Flow

1
User Interaction — FilterBar.vue

User picks a value from one of the 4 filter dropdowns: Time Period, Location (warehouse), Category, or Order Status. Each dropdown is a v-model binding.

reactive ref update
2
Global Filter State — useFilters.js

A singleton composable holds selectedPeriod, selectedLocation, selectedCategory, selectedStatus as reactive refs. All views share the same state object — changing a filter in the nav bar affects every page simultaneously.

watch / computed triggers re-fetch
3
API Client — api.js (Axios)

The view calls e.g. api.getOrders(filters). Filters with the value "all" are omitted from the request so the backend applies no restriction for that dimension. The remaining params become a URL query string: ?warehouse=London&status=processing.

HTTP GET over localhost
4
FastAPI Route Handler — main.py

FastAPI parses query params into typed Python arguments. The handler calls apply_filters(data, warehouse, category, status) then filter_by_month(data, month). Quarter values like Q1-2025 are expanded to individual month strings before filtering.

Python list comprehension on in-memory list
5
Mock Data Layer — mock_data.py

On server startup, mock_data.py reads all 7 JSON files into memory. Filtering always operates on the full in-memory dataset — there is no caching or incremental loading. No writes to disk occur during the session.

Pydantic serializes → JSON response
6
Vue Rendering — View Component

Axios resolves the promise. The view stores the result in a reactive ref. A v-for loop renders rows or cards. Some views add a secondary client-side computed filter (e.g. text search on Inventory). Monetary values are formatted via utils/currency.js.

Design Notes

Global filter singleton useFilters() uses module-level refs so every view shares the same reactive state — changing a filter from any page updates all pages at once.
No persistence All data is read from JSON at startup and held in memory. Writes (new purchase orders) are in-memory only and reset on server restart.
Filter semantics Filters with value "all" are not sent as query params; the backend interprets a missing param as "no restriction". Case-insensitive matching handles frontend-lowercase vs backend-titlecase status values.
CORS open in dev The backend allows all origins. This is intentional for local development and must be restricted before any deployment.
No chart library All charts are custom SVG elements and CSS Grid layouts. This keeps the bundle lean but means chart code lives directly in .vue components.
API docs FastAPI auto-generates interactive docs at localhost:8001/docs — useful for manual endpoint testing without a frontend.
